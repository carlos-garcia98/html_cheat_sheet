HTML Tags Cheat Sheet

This cheat sheet list some HTML tags that can be useful, they are grouped by function.

The information shown about the tags was retrieved from the MDN Web Docs. These are not all the tags that exist, to see more tags and their information check the MDN: HTML elements reference.

Document Metadata

Tag Description Self-closing?
<head> The head tag contains metadata (machine-readable information) about the document. No
<link> The link tag specifies relationships between the document and an external resource Yes
<meta> The meta tag represent the metadata taht cannot be represented by other HTML meta-related tag. Yes
<title> The title tag defines the document's title and it's shown on the browser's title tab or page's tab. No

Sectioning Root

Tag Description Self-closing?
<body> The body tag represents the content of an HTML document. There can be only one body tag in a document. No

Content Sectioning

Tag Description Self-closing?
<header> The header tag represents introductory content, typically a group of introductory or navigational aids. No
<main> The main tag represents the dominant content of the body document. No
<footer> The footer tag typically contains information about the author of the section, copyright data or links to related documents. No
<section> The section tag represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it. No
<article> The article tag represents a self-contained composition in a document, page, application or site, which is intended to be independently distributable or reusable. No
<aside> The aside tag represents a portion of a document whose content is only indirectly related to the document's main content. No
<h1>, <h2>, <h3>, <h4>, <h5>, <h6> The h1 to h6 tags represents six levels of sectioning headings. The h1 is the highest section level and h6 is the lowest. No
<nav> The nav tag represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. No

Text Content

Tag Description Self-closing?
<div> The div tag is the generic container for flow content. It has no effect in the content or the layout until styled in some way using CSS. No
<p> The p tag represents a paragraph. Paragraphs are usually represented in visual media as blocks of text separated from adjacent blocks by blank lines and/or first-line indentation. No
<ul> The ul tag represents an unordered list of items, typically rendered as a bulleted list. No
<ol> The ol tag represents an ordered list of items, typically rendered as a numbered list. No
&lli> The li tag is used to represent an item list. It must be contained in a parent element: an ordered list (ol), an unordered list (ul) or a menu (menu) No
<figure> The figure tag represents self-contained content, potentially with an option caption, which is specified using the figcaption tag. No
<figcaption> The figcaption tag represents a caption or legend describing the rest of the contents of its parent figure tag. No

Inline Text Semantics

Tag Description Self-closing?
<a> The a (or anchor) tag, with its href attribute creates a hyperlink to web pages, files, email addresses, location in the same page or anything else a URL can address. No
<br> The br tag produces a line break in text. It is useful for writing a poem or an address, where the division of lines are significant. Yes
<meta> The meta tag represent the metadata taht cannot be represented by other HTML meta-related tag. Yes
<code> The code tag displays its content styled in a fashion intended to indicate that the text is a short fragment of computer code. No
<span> The span tag is a generic inline container for phrasing content, which does not inherently represent anything. No
<em> The em tag marks text that has stress emphasis. No
<strong> The strong tag indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type. No

Image and Multimedia

Tag Description Self-closing?
<image> The image tag embeds an image into the document. Yes
<audio> The audio tag is used to embed sound content in documents. It may contain one or more audio sources, represented using the src attribute or the source element: the browser will choose the most suitable one. No
<video> The video tag embeds a media player which supports video playback into the document. No
<map> The map tag is used with area elements to define an image map (a clickable link area). No
